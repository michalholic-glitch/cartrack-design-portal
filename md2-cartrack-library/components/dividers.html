Dividers

A divider is a thin line that separates and groups content — between list items, sections of a form, or regions of a dense layout. Used sparingly, it brings order without adding weight.

Category: Data displayDesktopImport: Divider
Reach for whitespace and grouping first; add a divider only where a clear boundary helps. Too many lines make a layout feel busy.
No standalone MDC Web component. MDC Web has no dedicated divider component — it provides list dividers (mdc-deprecated-list-divider) and you draw a 1px rule using the divider colour token. The preview below is a Cartrack-styled approximation.

Anatomy

A single hairline rule.

  1. Line — 1px rule in the divider colour.
  2. Inset — optional left/right indentation to align with content (e.g. past an avatar).

Variants

VariantWhen to use
Full-widthSeparate major sections edge to edge.
InsetBetween list items, aligned past the leading element.
MiddleIndented on both sides for a lighter separation.
VerticalSeparate inline items in a toolbar.

Behaviors

Grouping

Use a divider to mark a real boundary between groups, not between every item — rows already separate visually.

Inset alignment

Inset dividers align with the text column so the eye reads a clean left edge.

States

StateTreatment
Default1px line at the divider colour (rgba(0,0,0,.12)).

Specs

PropertyValue
Thickness1px
Colourrgba(0,0,0,.12) (divider token)

Do & Don't

Do

  • Use dividers to mark real group boundaries.
  • Inset to align with the content column.
  • Prefer spacing where it reads clearly without a line.

Don't

  • Don't put a divider between every list row.
  • Don't use heavy or coloured rules for plain separation.
  • Don't rely on a divider to convey meaning by itself.

Accessibility

Decorative dividers are hidden from assistive tech; a divider that genuinely separates groups can use a separator role. Never use a line as the only way to convey structure — pair with headings/labels.
