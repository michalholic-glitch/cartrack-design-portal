TextField

Inputs & forms stable AA reviewed

Text fields let operators enter and edit text — a registration number, a driver's name, an odometer reading, a note. They come in filled and outlined styles and always carry a label so the field's purpose stays visible once filled.

Source: Material Components Web (MDC) 14.0.0 — Cartrack-themed · component-specs/text-fields.md
Pick one style and use it consistently across a form. In the Cartrack portal, outlined is the default for forms; filled suits dense, list-like settings.
When to use
  • Dense, settings-style lists; strong visual grouping.
  • Default for forms; clear field boundaries.
  • Search (leading magnifier + trailing clear), password (visibility toggle), units (trailing "km").
  • A field that opens a menu to choose one value.
  • Longer notes; grows with content.
When not to use
  • Don't use placeholder text as the only label.
Anatomy
  1. Container — filled (shaded box, bottom line) or outlined (border box).
  2. Label — floats above the value; stays visible when filled.
  3. Input text — the value the operator types.
  4. Supporting text — helper or error message below the field.
  5. Leading / trailing icon — optional adornment (search, clear, visibility, unit).
  6. Activation line / border — turns primary on focus, error on invalid.
Variants
FilledDense, settings-style lists; strong visual grouping.

Select / dropdown fieldA field that opens a menu to choose one value.
Multiline / textareaLonger notes; grows with content.
States
EnabledDefault container; medium-emphasis label.
HoverBorder/line darkens slightly.
FocusedPrimary 2px border (outlined) or 2px underline (filled); primary label.
ErrorError colour border/line, label, and supporting text.
Disabled38% opacity; not focusable.
Behaviors
Label floatThe label sits in the field when empty and floats up on focus or when a value is present, so it never hides the content.
ValidationValidate on blur or submit; show a concise error in the supporting text and turn the field's line/border to the error colour. Keep the label.
Read-only & disabledRead-only shows a value that can't be edited but can be copied; disabled dims the whole field and removes it from tab order.
Do
  • Always show a label; keep it visible when filled.
  • Write specific, helpful error text ("Enter a value", not "Invalid").
  • Use one field style per form.
  • Right-size width to the expected content.
Don't
  • Don't mix filled and outlined in the same form.
  • Don't rely on colour alone for errors — include text.

Accessibility
  • Every field has a programmatic <label>; errors are linked via aria-describedby and announced.
  • Focus is clearly visible.
  • Required and invalid states use aria-required / aria-invalid, not colour alone.
