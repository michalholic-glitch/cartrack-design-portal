ExpansionPanel

Surfaces stable AA reviewed

Expansion panels (accordions) let operators reveal detail on demand — collapsing a long record or settings page into named sections that open one at a time, so the screen stays manageable.

Source: Material Components Web (MDC) 14.0.0 — Cartrack-themed · component-specs/expansion-panels.md
Use expansion panels to tame dense detail (vehicle specs, settings groups). For switching between peer views of one record, prefer tabs.
When to use
  • Only one section open at a time (guided reading).
  • Several sections open at once (reference scanning).
  • Show a key value on the collapsed header.
When not to use
  • Don't use accordions where tabs fit better.
Anatomy
  1. Header — the always-visible section title + expand/collapse affordance.
  2. Expand icon — chevron indicating open/closed.
  3. Panel body — the content revealed when expanded.
  4. Summary meta — optional preview value shown on the collapsed header.
Variants
Single-openOnly one section open at a time (guided reading).
Multi-openSeveral sections open at once (reference scanning).
With summaryShow a key value on the collapsed header.
States
CollapsedHeader only; chevron pointing right/down.
ExpandedBody visible; chevron rotated.
Header hover/focusState layer; visible focus.
Behaviors
Expand/collapseClicking the header toggles the panel; the chevron rotates to reflect state.
Single vs multiDecide whether opening one closes others; keep it consistent within a group.
Deep linkingA section can open automatically when linked to, so operators land on the right detail.
Do
  • Name sections clearly so collapsed headers are scannable.
  • Show a summary value where it helps.
  • Keep single-vs-multi behavior consistent.
Don't
  • Don't nest accordions deeply.
  • Don't hide critical, always-needed info behind a collapse.

Accessibility
  • The header is a button exposing aria-expanded and controlling its panel via aria-controls.
  • It's keyboard operable with visible focus.
  • The chevron is decorative — state is conveyed programmatically.
