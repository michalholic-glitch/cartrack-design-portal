# cartrack-ai-design-system (source repo)

This repo has two independent halves. Which one you want depends on what you're doing.

## `design-system/` — the package

Everything an AI agent or developer needs to generate UI with this system: tokens,
components, page templates, the AI behavior contract, and a test harness that checks
the docs actually work. Self-contained — no dependency on anything else in this repo.

**Download or connect this folder.** See `design-system/README.md` and
`design-system/CLAUDE.md`.

## `portal/` — the documentation website

The public docs site (generated by `portal/docs/build_portal.py` into `portal/docs/`,
deployed to Vercel), the Cartrack *marketing* brand guidelines (`portal/brand/` —
reference-only, not part of the product design system), and the planning specs behind
past/future portal redesigns (`portal/Product Documentation/`).

This half exists to publish and explain the package above — it isn't needed to use it.

## If you're not sure which you need

Building UI, or setting up an AI coding tool to use this design system → `design-system/`.

Working on the docs website itself, its IA, or the marketing brand reference → `portal/`.

## A note on how this split works

`portal/docs/build_portal.py` reads directly from `design-system/` and regenerates its
own copies of the visual preview and the downloadable files/zip on every run — so the
package is always the single source of truth; the portal mirrors it, never the reverse.
Re-run the generator after changing anything in `design-system/` that the site reflects
(tokens, component docs, templates).
