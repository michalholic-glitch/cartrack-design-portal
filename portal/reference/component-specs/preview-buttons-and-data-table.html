Buttons

Status: stable Category: Actions MDC: mdc-button buttons.md

Buttons let operators trigger actions — saving a vehicle, running a report, confirming a change. Emphasis communicates priority: the one most important action on a view is the contained button; everything else steps down.

Context included in buttons.md

This is the structured guidance the AI uses to decide when and how to use the component.

Key rule: Use exactly one contained (high-emphasis) button per view. The FAB is excluded — it's a mobile pattern; on desktop the primary action sits in the page toolbar / app bar.

When to use

  • Contained — the single primary action (Save, Add, Run).
  • Outlined — important secondary actions, or Cancel beside a contained button.
  • Text — low-emphasis actions in toolbars, cards, dialog dismissals.
  • Toggle — switch between exclusive views (Map / List); 2–5 options, not navigation.

When NOT to use

  • Not for navigation that's really a link → use a Link.
  • Not stacked as multiple contained buttons.
  • Not white-on-orange for important labels (fails AA).

Contained fill / labelprimary / white — use primary-dark (#BB4800) where white text must pass WCAG AA (white on #F47735 is only ~2.79:1)

Do

  • Keep one contained action per view.
  • Use verb-first labels ("Add vehicle", not "New").
  • Pair Cancel (outlined/text) beside the primary.

Don't

  • Don't stack multiple contained buttons.
  • Don't ship white-on-orange for important labels.
  • Don't hard-code colours/sizes — use tokens.

Correct usage (themed MDC markup)

<!-- one primary (raised) action, secondary as outlined -->
<button class="mdc-button mdc-button--raised">
  <span class="mdc-button__ripple"></span>
  <span class="mdc-button__label">Save vehicle</span>
</button>
<button class="mdc-button mdc-button--outlined">
  <span class="mdc-button__ripple"></span>
  <span class="mdc-button__label">Cancel</span>
</button>

Accessibility

  • Real <button> elements with discernible text; icon-only actions get an aria-label.
  • Focus is always visible; disabled state is conveyed to assistive tech.

Variants

Context included in data-table.md

This is the structured guidance the AI uses to decide when and how to use the component.

Key rule: When a dataset is large, server-paginated, sortable, or selectable, use the data table — not a basic Table. Reserve the basic table for small, static key/value layouts.

When to use

  • With toolbar — almost always; title + search/filter/export.
  • Selectable — bulk actions on multiple records.
  • Paginated — any dataset beyond one screen.
  • Dense / inline-edit / expandable — high-volume or detail-rich screens.

When NOT to use

  • Not infinite-scroll on desktop → paginate.
  • Not multi-column sort unless explicitly needed.
  • Not status by colour alone → keep the text label.

Behaviors

  • Sorting — click a header to sort; click again to reverse; one column at a time.
  • Selection — row checkboxes + select-all (indeterminate when partial); toolbar switches to a contextual toolbar showing count + bulk actions.
  • Pagination — footer with rows-per-page + range; fetch per page for server data.

Specs

Do

  • Right-align numeric columns with tabular figures.
  • Map status to consistent colour chips (Active/Idle/Offline).
  • Paginate; fetch per page for large data.

Don't

  • Don't bury constantly-used actions in a row overflow menu.
  • Don't infinite-scroll a desktop data table.
  • Don't encode status by colour alone.

Correct usage (themed MDC markup)

<div class="mdc-data-table">
  <table class="mdc-data-table__table" aria-label="Vehicles">
    <thead><tr class="mdc-data-table__header-row">
      <th class="mdc-data-table__header-cell" scope="col">Registration</th>
      <th class="mdc-data-table__header-cell mdc-data-table__header-cell--numeric">Odometer (km)</th>
    </tr></thead>
    <tbody class="mdc-data-table__content">
      <tr class="mdc-data-table__row">
        <th class="mdc-data-table__cell" scope="row">CA 123-456</th>
        <td class="mdc-data-table__cell mdc-data-table__cell--numeric">84,210</td>
      </tr>
    </tbody>
  </table>
</div>

Accessibility

  • Real <table> semantics with <th> headers; sortable headers expose aria-sort.
  • Select-all announces its indeterminate state; status is never colour-only.
