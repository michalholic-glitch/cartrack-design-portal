Breadcrumbs

Navigation stable AA reviewed

Breadcrumbs show where the operator is within the portal's hierarchy and let them step back up it — from a single vehicle, to the vehicle list, to the fleet. They orient users in deep desktop navigation.

Source: Material Components Web (MDC) 14.0.0 — Cartrack-themed · component-specs/breadcrumbs.md
Use breadcrumbs when content sits more than one level deep. They complement — not replace — the left navigation drawer.
When to use
  • Show the full path for shallow hierarchies.
  • Hide middle levels behind "…" for deep paths.
  • A crumb opens a menu of sibling pages.
When not to use
  • Don't use breadcrumbs as the primary navigation.
Anatomy
  1. Crumb link — an ancestor level (navigates up).
  2. Separator — a divider glyph between crumbs.
  3. Current page — the last crumb, not a link.
  4. Overflow — optional collapse ("…") for very deep paths.
Variants

Demos are the real preview markup, inlined at build time. "Example usage" lines are generated from doc.json — a documentation aid, not tested code.

States
LinkOn-surface-medium; underlines on hover.
CurrentOn-surface, not a link.
FocusVisible focus on each link.
Behaviors
NavigationEach crumb except the last navigates to that ancestor; the current page is plain text.
TruncationLong labels truncate with a tooltip; deep paths collapse the middle.
Do
  • Make the current page the non-link last crumb.
  • Collapse very deep paths.
  • Mirror the real IA hierarchy.
Don't
  • Don't show a one-level trail (no value).
  • Don't make the current page a link to itself.

Accessibility
  • Wrap in a nav labelled "Breadcrumb" with an ordered list.
  • The current page uses aria-current="page".
  • Separators are decorative (hidden from assistive tech).
  • Links are keyboard reachable with visible focus.
